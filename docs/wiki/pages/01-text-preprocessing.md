# 文本预处理

## 概述

文本预处理是构建三国知识库的基础环节，目标是将《三国志》《三国演义》《后汉书》《资治通鉴》等古籍原文转化为结构化、可处理的文本数据。本流程涵盖文本清洗、篇章分段、句级分割、实体预标注四个核心步骤，为后续实体提取、事件抽取等任务奠定数据基础。

## 输入

- 《三国志》（陈寿撰，裴松之注）原文TXT/PDF
- 《三国演义》（毛评本）原文TXT
- 《后汉书》《资治通鉴》相关卷章
- 辅助参考资料：卢弼《三国志集解》、中华书局点校本

## 处理流程

### 1. 文本清洗

- **乱码处理**：检测并修复GB2312/UTF-8编码混用问题，使用编码检测工具（chardet）自动识别
- **标点标准化**：将全角/半角标点统一为中文全角标点（，。——？！等）
- **注释分离**：裴松之注文以"臣松之案"标记开头，需与正文分离存储，保留关联关系
- **避讳字还原**：清代避讳字（如"玄→元""弘→宏"）按朝代还原
- **异体字统一**：建立异体字映射表（如"峣→跷""宻→密"），统一为标准字形

### 2. 篇章分段

- **纪传体分段**：按本纪、列传、志、表四类分段，每段保留元数据（卷号、篇名、作者）
- **章回体分段**：《三国演义》按120回自然分段，保留回目名称
- **段落边界检测**：以"卷X""第X回""臣XX曰"等为标记，结合空行判断段落边界
- **段落元数据**：每段标注来源、卷号、起始行号、字符数

### 3. 句级分割

- **古文句读**：基于古籍句读标记（。！？；）进行初分，结合规则处理引文嵌套
- **人名/地名边界保护**：建立专名词典，防止"关羽""张飞"等专名被错误切断
- **对话识别**：识别"曰""云""谓"后的直接引语，独立成句
- **长句拆分**：对超过100字的句子，按"而""则""以"等连词做语义切分

### 4. 实体预标注

- **规则匹配**：基于专名词典（人物表、地名表、官职表）进行首次匹配标注
- **标注格式**：采用BIO标注格式（B-实体开始、I-实体内部、O-非实体）
- **质量检查**：随机抽样5%检测标注一致性，要求F1值>0.95

## 输出

- 清洗后的纯文本文档，UTF-8编码，每行一句
- 分段元数据JSON文件（段落ID、来源、卷号、起止行号、字符数）
- 预标注BIO格式文件，可供NER模型直接训练

## 常见问题

1. **编码乱码**：古籍扫描件OCR后常见GB18030编码，需用encoding-detector检测后统一转换
2. **引文嵌套**：正文引裴注、裴注又引他书，需用栈结构维护引文层级
3. **断句争议**："XX曰"后内容有时是叙述而非对话，需结合上下文判断
4. **文本缺失**：部分卷章有脱文，标记为[MISSING]并在元数据中记录
