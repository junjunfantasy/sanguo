# 事件提取

## 概述

事件提取是从文本中识别、分类和结构化历史事件的核心流程。三国领域定义了12类事件类型，涵盖军事、政治、人事三大维度。每个事件需提取触发词、参与者、时间、地点、结果五要素，并完成时间归一化处理，为后续知识图谱提供事件节点。

## 输入

- 已标注实体的三国语料（含人物、地名、时间等20类实体）
- 实体链接表（实体ID→标准化名称映射）
- 三国历史年表（《资治通鉴》卷次对照表）

## 处理流程

### 1. 事件类型定义（12类）

| 编号 | 事件类型 | 说明 | 触发词示例 | 要素 |
|------|---------|------|-----------|------|
| EV01 | 军事冲突(MilitaryConflict) | 战役/战斗/攻守 | 征、伐、讨、攻、战、破、击 | 进攻方、防守方、地点、胜负 |
| EV02 | 政治决策(PoliticalDecision) | 诏令/禅让/变法 | 诏、令、制、敕、废、立 | 决策者、内容、目的 |
| EV03 | 人事任免(Appointment) | 拜官/迁转/罢黜 | 拜、迁、谪、免、举、征 | 人物、官职、任免类型 |
| EV04 | 外交活动(Diplomacy) | 会盟/使节/和亲 | 盟、会、遣使、朝贡、和亲 | 派遣方、接收方、目的 |
| EV05 | 自然灾害(Disaster) | 旱/蝗/地震/瘟疫 | 旱、蝗、地震、大疫、饥 | 时间、地点、灾种、影响 |
| EV06 | 人事变动(LeadershipChange) | 即位/驾崩/废黜 | 即位、崩、薨、卒、废 | 人物、新身份、前任 |
| EV07 | 法律刑罚(LegalAction) | 立法/判刑/赦免 | 法、刑、诛、赦、斩 | 施刑者、受刑者、罪行 |
| EV08 | 经济措施(EconomicAction) | 屯田/征税/铸币 | 屯田、课税、铸钱、开仓 | 执行者、措施内容、区域 |
| EV09 | 文化行为(CulturalEvent) | 著述/建学/祭祀 | 著、撰、修、祭、封禅 | 人物、作品/活动、时间 |
| EV10 | 家族事件(ClanEvent) | 婚嫁/出生/死亡 | 娶、嫁、生、卒、葬 | 相关人物、关系变化 |
| EV11 | 谋略计策(PlotScheme) | 离间/伏击/诈降 | 计、谋、间、诈降、诈败 | 策划者、目标、方式 |
| EV12 | 领土变更(TerritoryChange) | 割地/入附/迁移 | 割、让、附、徙、迁都 | 让与方、接收方、区域 |

### 2. 事件触发词识别

- **规则匹配**：基于事件类型关键词表（约500个触发词），使用正则+词典匹配
- **触发词扩展**：基于同义词扩展（征=伐=讨=击），构建触发词同义网络
- **AI辅助**：使用BERT分类模型判断含触发词的句子是否构成事件实体，防止误匹配

### 3. 事件要素抽取

- **参与者抽取**：从事件句的语法依存关系中提取主语（主动方）和宾语（被动方）
- **时间抽取**：提取句中的年号时间、干支时间、相对时间（"明年""初"），传递至时间归一化模块
- **地点抽取**：提取事件发生地点，关联空间归一化模块
- **结果判定**：基于语义分析判断胜负（军事事件）或成败（计谋事件），标记为WON/LOST/UNKNOWN
- **置信度评估**：基于要素完整度（5要素中抽取N个）计算事件置信度

### 4. 事件时间归一化

- **年号转公元**："建安五年"→ 200年，"建安十三年"→ 208年
- **干支转换**："甲子年"按60年周期匹配最近事件所在周期
- **季节映射**："春→3月、夏→6月、秋→9月、冬→12月"
- **相对时间**："明年→事件年份+1"，"初（年号）→该年号起始年"
- **不确定处理**：多个可能时间取概率最大的，标注置信度

## 输出

- 事件JSON文件：每条事件含ID、类型、触发词、5要素（参与者/时间/地点/结果/置信度）
- 事件-实体关联表：事件涉及的人物、地点、势力列表
- 时间归一化映射表：原文时间→公元时间的转换记录

## 常见问题

1. **事件重叠**：同一句描述多个事件（"曹操攻徐州，刘备奔冀州"），需按分句拆分
2. **隐含事件**：无明确触发词但包含事件语义的句子，使用AI分类器补充
3. **时间缺失**：部分事件上下文推断时间，需从相邻段落继承
4. **结果争议**：官渡之战等结果无争议，但"诸葛亮北伐"可视为成功/失败的不同解读
