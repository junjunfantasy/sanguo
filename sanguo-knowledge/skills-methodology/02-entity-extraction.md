# 实体提取

## 概述

实体提取是将非结构化文本中的特定实体识别并归类，是知识图谱构建的核心环节。本流程针对三国领域定义了20类实体类型，采用"规则预匹配+词典辅助+AI模型精标"的三层流水线策略，兼顾准确率与召回率。

## 输入

- 预处理后的三国文本语料（每行一句，BIO格式标注）
- 三国专名词典库（人物、地名、官职、器物、典故等）
- 《三国志》索引表、《中国历史大辞典·三国卷》辅助资料

## 处理流程

### 1. 实体类型定义（20类）

| 编号 | 实体类型 | 类型说明 | 示例 |
|------|---------|---------|------|
| E01 | 人物(Person) | 历史人物 | 曹操、诸葛亮、关羽 |
| E02 | 地点(Location) | 州郡城池山川关隘 | 许都、赤壁、荆州 |
| E03 | 势力(Faction) | 政权/军事集团 | 魏、蜀汉、东吴 |
| E04 | 官职(Official) | 官名/爵位 | 丞相、大将军、亭侯 |
| E05 | 年号(EraName) | 帝王年号 | 建安、黄初、章武 |
| E06 | 武器(Weapon) | 兵器/装备 | 青龙偃月刀、方天画戟 |
| E07 | 器物(Artifact) | 宝物/信物 | 玉玺、赤兔马、木牛流马 |
| E08 | 战役(Battle) | 军事战役/战斗 | 官渡之战、赤壁之战 |
| E09 | 典籍(Document) | 书籍/文章 | 《孟德新书》《出师表》 |
| E10 | 制度(Institution) | 制度/法规 | 屯田制、九品中正制 |
| E11 | 事件(Event) | 历史事件 | 衣带诏、魏王篡汉 |
| E12 | 时间(Time) | 时间表达 | 建安五年、初平元年春 |
| E13 | 军事单位(MilitaryUnit) | 军队/编制 | 虎豹骑、青州兵、水师 |
| E14 | 策略(Strategy) | 计谋/策略 | 连环计、空城计、苦肉计 |
| E15 | 亲属关系(Kinship) | 亲属称谓 | 兄、弟、子、母、妻 |
| E16 | 城防(Fortification) | 城墙/关隘 | 虎牢关、函谷关、剑阁 |
| E17 | 行政区划(AdminRegion) | 州/郡/县 | 司隶州、豫州、颍川郡 |
| E18 | 山水(NaturalFeature) | 山河湖海 | 长江、太行山、洞庭湖 |
| E19 | 宗教(Religion) | 信仰/祭祀 | 黄天、五斗米道 |
| E20 | 谥号(PosthumousTitle) | 谥号/庙号 | 武帝、昭烈帝、文皇帝 |

### 2. 规则预匹配（第一阶段）

- **基于正则的确定性匹配**：年号（建安\d+年）、谥号（某皇帝）、官职（大[将军司马]）等模式稳定
- **核心词典匹配**：使用AC自动机（Aho-Corasick）算法进行多模式匹配，支持模糊匹配（编辑距离≤2）
- **上下文消歧**："荆州"可指州或治所城市，"丞相"在不同时期指代不同人物，需上下文规则裁决

### 3. 模型精标（第二阶段）

- **模型选择**：基于RoBERTa-wwm-ext预训练模型，微调序列标注层
- **训练数据**：人工标注5000句作为种子数据，使用远程监督（Distant Supervision）扩充至50000句
- **解码策略**：CRF条件随机场层约束标签转移（B-Person后不能接I-Location）
- **模型集成**：3个不同随机种子的模型投票，多数决决定标注结果

### 4. 后处理与合并

- **跨段合并**：同一段落中"曹操""曹孟德""魏武"合并为同一实体ID
- **实体消歧**：同名人物（如两个"张鲁"）、同名地名（"东郡"在不同朝代不同）的上下文消歧
- **置信度过滤**：仅保留模型置信度≥0.85的标注结果

## 输出

- 实体标注JSON文件：每句一个对象，含实体列表（类型、文本、起始位置、置信度、实体ID）
- 实体统计报告：各类型实体数量、覆盖度、F1值
- 实体链接表：实体文本→标准化实体ID的映射

## 常见问题

1. **人名歧义**：同人异名（诸葛亮=孔明=卧龙）、同名异人（张鲁有两人），需建立别名词典和上下文消歧规则
2. **官职变迁**：同一人物在不同时期的官职不同，需标注时间范围
3. **地名演变**：同一地名在不同时期指代区域不同，需后续空间归一化模块处理
4. **嵌套实体**："荀彧举荐荀攸"中涉及两个人物，需保证同时识别
